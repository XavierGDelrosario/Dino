# docs/audit

Compliance & posture audits — point-in-time checks that DINO is meeting its obligations
(data licenses, attribution, security, privacy, dependencies). Distinct from
`docs/research/` (which explores open questions): an audit **verifies a known standard is
being met** and lists any gaps to close.

**What goes here:** data/dependency license audits, attribution-coverage checks, security
review summaries, privacy/PII inventories, pre-launch legal checklists. Each audit is
dated and states what was checked, the standard, findings, and open gaps.

**Cross-refs:** the source of truth for data licenses is [`ATTRIBUTION.md`](../../ATTRIBUTION.md);
security tooling is `/security-review`; the launch checklist is `docs/TODO.md`.
